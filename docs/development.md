# Local Development Guide

이 문서의 가이드는 파이썬 패키지 및 가상환경 관리 도구인 `uv`를 기준으로 작성되었습니다.

## uv 설치

우분투(Linux) 환경에서 아래 명령어를 실행하여 `uv` 독립형 바이너리를 설치합니다.

```bash
curl -LsSf https://astral.sh/uv/install.sh | sh
```

설치 후 터미널을 재시작하거나 `source ~/.bashrc` (또는 사용하는 쉘 설정 파일)를 실행하여 환경 변수를 반영합니다.

## 가상환경 생성 및 활성화

프로젝트 루트 디렉토리에서 작업을 진행합니다.

### 가상환경 생성
```bash
uv venv
```
* 명시적인 파이썬 버전을 지정하려면 `uv venv --python 3.12` 형태로 실행합니다.

### 가상환경 활성화
```bash
source .venv/bin/activate
```
* 활성화되면 터미널 프롬프트 왼쪽에 `(.venv)` 표시가 나타납니다.

## 의존성 패키지 관리

`pyproject.toml`을 기반으로 패키지를 추가하고 관리합니다.

### 일반(프로덕션) 의존성 추가
런타임에 반드시 필요한 패키지(예: 시리얼 통신, MQTT)를 추가합니다.
```bash
uv add pyserial paho-mqtt
```

### 개발용(Development) 의존성 추가
테스트나 코드 포맷팅 등 개발 과정에서만 사용하는 패키지는 `--dev` 플래그를 붙여 분리합니다.
```bash
uv add --dev ruff pytest
```

### 기존 환경 동기화 (전체 설치)
레포지토리를 새로 클론 받았거나 `pyproject.toml` 변경 사항을 로컬 환경에 그대로 반영할 때는 아래 명령어를 사용합니다. 일반 의존성과 개발용 의존성이 모두 자동으로 설치됩니다.
```bash
uv sync
```

## 코드 포매팅 및 린트 (Ruff)

이 프로젝트는 매우 빠른 파이썬 코드 포매터이자 린터인 ruff를 사용합니다. 코드를 수정한 후 커밋하기 전에 아래 명령어들을 사용하여 코드 스타일을 일관되게 정리할 수 있습니다.

> 아래 명령들은 `.venv`가 활성화된 상태를 전제로 합니다. 시스템 파이썬으로 실행하지 마세요.

### 코드 포매팅 (코드 자동 정렬)
```bash
ruff format
```

### 코드 린팅 (문법 오류 및 안티 패턴 검사)
```bash
ruff check --fix
```

## pre-commit 훅 설정

일관된 코드 스타일 유지와 정적 분석을 위해 커밋 전에 자동으로 코드를 검사하는 pre-commit 훅을 설정합니다. 프로젝트 루트 디렉토리에서 아래 명령어를 실행하세요.
```bash
pre-commit install
```
* 설치가 완료되면 이후 git commit을 실행할 때마다 .pre-commit-config.yaml에 정의된 린터와 포매터(예: ruff)가 자동으로 작동하여 코드를 검사합니다.

### 전체 파일 수동 검사 (Manual Run)
git commit을 실행하지 않고 명시적으로 전체 파일에 대해 코드 검사와 포매팅을 일괄 수행하고 싶다면 아래 명령어를 사용합니다.
```bash
pre-commit run --all-files
```

## 도커 빌드 및 내부 구조 검증

로컬 우분투 환경에서 하드웨어 장치(USB)를 직접 연결하여 가동하지 않더라도, 도커 이미지가 에러 없이 빌드되는지 및 컨테이너 내부 공간(`/app`)에 파일이 의도한 구조로 올바르게 배치되었는지 검증하는 방법입니다.

### 도커 이미지 빌드
프로젝트 루트 디렉토리에서 아래 명령어를 실행하여 도커 빌드가 정상적으로 수행되는지 확인합니다.
```bash
docker build -t wallpad-mqtt-bridge .
```

### 컨테이너 내부 파일 구조 확인 (인터랙티브 쉘 진입)
빌드가 완료된 이미지 내부의 파일 시스템을 직접 눈으로 확인하기 위해, 메인 스크립트(`run.sh`) 대신 컨테이너의 `bash` 쉘을 실행하여 진입합니다.
```bash
docker run --rm -it wallpad-mqtt-bridge bash
```
* `--rm`: 검증을 마치고 컨테이너를 종료할 때, 잔여 컨테이너 찌꺼기를 자동으로 삭제합니다.
* `-it`: 터미널 환경에서 대화형 인터페이스로 컨테이너 내부 시스템과 통신합니다.

### 내부 검증 명령어
컨테이너 내부에 진입하면 프롬프트가 바뀝니다. 아래 명령어들을 차례로 입력하여 구조를 검증합니다.
```bash
# 1. 현재 작업 디렉토리 확인 (Dockerfile에서 설정한 /app이 나와야 함)
pwd

# 2. 파일 목록 확인 (*.py 파일들과 run.sh만 깔끔하게 들어있어야 함)
ls -la

# 3. .dockerignore 작동 여부 확인 (.venv, docs/, tests/ 등이 복사되지 않고 잘 제외되었는지 확인)
# ls -la 명령 결과에 해당 폴더들이 보이지 않아야 정상입니다.
```

검증이 모두 끝났다면 `exit`를 입력하여 로컬 호스트 터미널로 복귀합니다.
```bash
exit
```
